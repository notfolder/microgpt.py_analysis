# 共通ガイドライン
- ユーザーへの回答と設計書、コード上のコメントは基本的には日本語にしてください
- llmへ渡すプロンプトは**必ず**英語にしてください

# 1. ドキュメント作成モード
ドキュメント作成（README, API仕様書, AGENTS.md 自体の更新）を依頼された場合は以下のルールに従ってください。
- 仕様書/設計書を書く際には**コードは絶対**に書かないでください。日本語で処理を具体的にに記述すること(設定例や仕様を説明するためのyamlやjson形式の例は記述して良い)
- 仕様書/設計書を書く際には将来の拡張案など、仕様が曖昧になる記述は絶対に記述しないこと
- 仕様書/設計書を書く際には実装スケジュール/実装フェーズは絶対に記述しないこと
- 仕様書/設計書はmarkdown形式で記述し、複雑な説明にはmermaid形式の図を描くようにすること
- 仕様書/設計書を作成,修正したら**絶対に、必ず**内容をレビューして内容に**実装にあたって曖昧な点はないか(「または」表現等、実装オプションを提示しているなどがないか)**、**矛盾はないか**、**冗長ではないか**、**技術的な無理はないか**を確認、修正して**必ず**レビュー結果を報告してください
- 画面のwireframeはascii artで書いてください。画面のwireframe以外の図はmermaid形式で書いてください

# 2. コード実装モード
実装を依頼された場合は以下のルールに従ってください。
- コーディング時はできるだけ日本語でコメントをつけて
- コードの作成を依頼されたら、コードの省略は絶対せず、動作するのに足りるコードを作成し切ってください
- コードを作成,修正したら、**絶対に、必ず**自己レビューをしてユーザーに報告してください
- 自己レビューの時には関連するコードを必ず読んで、***より良い設計がないか**,**関連コードに与える影響**、**冗長なコードがないか**を確認、修正して**必ず**レビュー結果を報告してください
- 良い設計とは、シンプル,バグが少ない,わかりやすい,技術的な無理がない設計です
- pythonの実装時はできるだけPEP8に準拠して,型ヒントもできるだけつける様にして,docstring形式のコメントは必ずつけて
